🏗️ 04. Model Building Pipeline

💡 Approach: We utilize Contrastive Language-Image Pre-training (CLIP) to perform classification without traditional retraining, comparing Zero-shot and Few-shot capabilities.

CLIP uses dual encoders (a Vision Transformer for images and a Text Transformer for captions) to map both modalities into a shared high-dimensional embedding space.

🔄 Data Pipeline & Tensor Shapes

1. Image Pipeline: (B, 3, 224, 224) ➔ Image Encoder ➔ (B, 512) // L2 Normalized
2. Text Pipeline: (Num_Classes, 77) ➔ Text Encoder ➔ (Num_Classes, 512) // L2 Normalized

🧠 Classification Head (Similarity Matching)

Unlike traditional linear heads, CLIP performs classification via Cosine Similarity (Dot Product) in the shared embedding space:
  • Zero-shot: Computes the dot product between Image Features (B, 512) and Text Features of prompts (5, 512), yielding probability logits: (B, 5).
  • Few-shot: Instead of text prompts, it computes similarity between the input Image Feature (B, 512) and the pre-extracted Class Prototypes (5, 512) from a few training images.
