🏗️ 04. Model Building Pipeline

💡 Approach: We utilize Contrastive Language-Image Pre-training (CLIP) to perform classification without traditional retraining, applying an aggregated Text Prototype method for Few-shot capabilities.

CLIP uses dual encoders (a Vision Transformer for images and a Text Transformer for captions) to map both modalities into a shared high-dimensional embedding space.

1. Text Prototype Aggregation (The Few-Shot Core)

Instead of matching against a single prompt, the model processes all available captions for a class. The text features are normalized, averaged together (Mean Aggregation), and normalized again to form a single, robust Class Prototype vector (5, 512).

# Create an aggregated representative vector for a class
for c in classes:
    # Extract and normalize text features for all captions of class c
    outputs = model.get_text_features(**inputs)
    text_features = outputs / outputs.norm(dim=-1, keepdim=True)
    
    # Aggregate N vectors into 1 single prototype via Mean
    class_vector_aggregated = torch.mean(all_text_features, dim=0)
    
    # Final normalization of the prototype
    class_vector_final = class_vector_aggregated / class_vector_aggregated.norm(dim=-1, keepdim=True)
    class_vectors[c] = class_vector_final

2. Classification via Similarity

When predicting a new image, its extracted feature vector (1, 512) is compared against the 5 pre-computed aggregated text prototypes using dot product (Cosine Similarity). The class with the highest similarity score is chosen.

🔍 Key Observations on Zero-Shot Errors

Without fine-tuning, CLIP occasionally relies on strong contextual biases rather than the main subject:

  • 🐄🐑
    Cow vs. Sheep Confusion Both animals frequently appear in similar rural contexts (green pastures, fences, farms). CLIP's attention maps often highlight the grassy background rather than the animal itself, leading to misclassification.
  • 🐕🐈
    Dog vs. Cat Overlap When photographed from a distance or on typical indoor furniture (like a sofa), the visual features of small dogs and cats heavily overlap in the embedding space.

🛡️ 07. Robustness Evaluation (Noise Data)

To evaluate the resilience of the CLIP architecture, we injected Gaussian noise into the test images prior to computing embeddings:

def add_noise(images, std=0.1):
    noise = torch.randn_like(images) * std
    noisy_images = images + noise
    return torch.clamp(noisy_images, -1, 1)
